# ADR-0001: Use MCP Protocol for AI Assistant Integration

**Date:** 2026-03-17
**Status:** Accepted

## Status

Accepted

## Context

ForgeCraft must deliver engineering standards to AI coding assistants (Claude, Copilot, Cursor, etc.)
at the point of code generation — not as a post-hoc review step. Three integration options were
evaluated: (1) publish instruction files the practitioner manually copies, (2) provide a REST API
the assistant polls, (3) implement Model Context Protocol (MCP) so assistants can call ForgeCraft
tools directly.

The key constraint is that the assistant must be able to invoke ForgeCraft actions during its
generation loop without human mediation. It must also be usable in CI/CD pipelines and as a CLI.

## Decision

Implement ForgeCraft as an MCP server exposing a unified router action (`forgecraft_router`) with
a single Zod-validated schema. The same codebase exports a CLI entry point (`npx forgecraft-mcp`)
that maps subcommands to the same handlers.

MCP was chosen because it is natively supported by Claude, Cursor, and VS Code Copilot without
additional tooling. The stdio transport requires no network configuration and works in all
environments. The unified router pattern means one schema file serves both MCP and CLI consumers.

## Alternatives Considered

- **REST API**: Requires the assistant to have network access and a running server. Adds operational
  complexity (auth, port management) without adding capability. Rejected.
- **Instruction files only (no tooling)**: Cannot enforce standards programmatically — relies on the
  assistant reading and following instructions. Experiments showed significant drift without
  automated feedback loops. Rejected.
- **One MCP tool per action**: 21+ tools in the MCP manifest creates token overhead per call and
  makes schema evolution harder. The unified router with a discriminated `action` field is more
  maintainable. Rejected.

## Consequences

- All assistant integrations go through MCP or CLI — no separate SDKs or REST endpoints.
- The unified router schema must remain backward compatible; new actions are additive.
- MCP server should be disabled after initial project setup to reclaim token budget; CLAUDE.md
  should direct Claude to use `npx forgecraft-mcp` CLI commands for subsequent operations.
- The stdio transport is not suitable for multi-tenant server use; a separate HTTP transport
  layer is needed for the 1.0 SaaS offering (see ADR-0003).
