# Specs

Product/use-case-driven specifications. One file per feature or major capability.

Each spec answers:
- **What** does the user do?
- **What outcome** do they want?
- **What is in scope / out of scope**?
- **Alternatives** considered

`feat:` and behavior-affecting `refactor:` commits should touch a spec here.

Legacy single-file specs live at `docs/PRD.md`, `docs/spec.md`, etc. — see `docs/manifest.yaml` for the full list. New specs go here.
