## Data Flow

```
user project
  └─ forgecraft-mcp (MCP tool, runs locally)
       ├─ fetchRemoteGates() ──────────────────────────────────────────────────►
       │    src/registry/remote-gates.ts                                        │
       │    24h TTL cache → .forgecraft/gates-cache.json                        │
       │    fallback: embedded templates if fetch fails                         │
       │                                                                        ▼
       │                                              quality-gates (GitHub repo)
       │                                              gates/**/*.yaml
       │                                              index.json (built by build-index.js)
       │                                                    ▲
       │                                                    │ raw.githubusercontent.com
       │                                                    │
       └─ contributeGates() ──────────────────────────────►│
            src/tools/contribute-gate.ts                   │
            reads .forgecraft/project-gates.yaml           │
            sends to forgecraft-server                      │
                  │                                         │
                  ▼                                         │
         forgecraft-server (Railway)                        │
         POST /contribute/gate                              │
         → creates GitHub issue in quality-gates ──────────┘
         → on approval: gate graduated to gates/ via GH Actions

genspec-portal (GitHub Pages)
  ├─ taxonomy.json (committed, updated on each forgecraft-mcp release via npm run export:taxonomy)
  └─ index.json ← fetched live from quality-gates on page load (no deploy needed for new gates)
```

## Quality Gates Registry

**Location**: `C:\workspace\genspec-dev-quality-gates\`

**Structure**:
```
gates/
  universal/          ← applies to all projects (mutation-testing, owasp-l1-static, dependency-audit)
  fintech/            ← financial/DeFi projects (15 gates from AEGIS session learnings)
  quarantine/         ← proposed gates pending review
index.json            ← generated by scripts/build-index.js, committed on every change
GATE_SCHEMA.md        ← schema reference for writing new gates
```

**Adding a gate**:
1. Create `gates/<tag>/<id>.yaml` following GATE_SCHEMA.md
2. Run `node scripts/build-index.js` to regenerate `index.json`
3. Commit and push — portal updates automatically on next page load
4. For community submissions: use `forgecraft contribute_gate` action

**Gate schema fields**: `id`, `title`, `description`, `category`, `gsProperty`, `phase`, `hook`,
`check`, `passCriterion`, `tags`, `generalizable`, `evidence`, `status`, `approvedAt`, `contributor`

**GS properties** (7): `verifiable`, `defended`, `derivable`, `completable`, `consistent`,
`self-correcting`, `accountable`

## forgecraft-server

**Location**: `C:\workspace\forgecraft-server\`
**Deploy**: Railway — `cd C:\workspace\forgecraft-server && railway up --detach`
**Env vars** (set in Railway dashboard or via `railway variables set`):
- `GITHUB_TOKEN` — GitHub PAT with `repo` scope, used to create issues in quality-gates
- `PORT` — defaults to 3000

**Routes**:
- `GET  /health` — status, version, timestamp
- `POST /contribute/gate` — Zod-validated gate submission (anonymous/attributed modes)
- `GET  /gates` — proxies quality-gates `index.json`
- `GET  /taxonomy` — serves `taxonomy.json` for portal

**Without `GITHUB_TOKEN`**: contributions return `202 pending` and are NOT lost —
forgecraft-mcp writes them to `.forgecraft/pending-contributions.json` for retry.

## Contribution Modes

Configured in project's `forgecraft.yaml`:
```yaml
contribute_gates: false        # default — nothing sent
contribute_gates: anonymous    # gate sent, no attribution
contribute_gates: attributed   # gate + GitHub handle, earns Pro credit on approval
```

Only gates with `generalizable: true` in `.forgecraft/project-gates.yaml` are submitted.
Already-submitted gates tracked in `.forgecraft/contributions.json` (no duplicates).

## Exceptions Mechanism

**File**: `.forgecraft/exceptions.json`
**Purpose**: Register false positives in pre-commit hooks so they are never re-discovered.
**Schema**: `{ version, exceptions: [{ id, hook, pattern, reason, addedAt, addedBy, adr? }] }`

Current exceptions:
- `hooks-yaml-token-pattern`: `templates/universal/hooks.yaml` triggers secrets scanner
  because it contains regex *patterns* for detecting tokens, not actual tokens.

## Portal Updates

The portal has two data sources:
1. **`taxonomy.json`** (committed in portal repo) — instruction blocks, hooks, archetypes.
   Update by running `npm run export:taxonomy` in forgecraft-mcp and committing to genspec-portal.
2. **quality-gates `index.json`** (fetched live) — community gates. Auto-updates on page load.
   No portal deploy needed when gates are added.
